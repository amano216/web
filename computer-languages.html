プログラミング言語の分類と仕組み

プログラミング言語の実行方式の違い、コンパイラとインタプリタの仕組み、各方式の特徴と用途を解説します。

プログラミング言語の実行方式

主な実行方式の分類

実行方式 英語名 代表的な言語 特徴
コンパイル型 Compiled Language C, C++, Go, Rust 事前に機械語に変換
高速実行・配布容易
インタプリタ型 Interpreted Language Python, Ruby, JavaScript, PHP 逐次解釈・実行
開発効率高・柔軟性
バイトコード型 Bytecode Language Java, C#, Python(一部) 中間コードに変換
仮想マシン上で実行
JIT型 Just-In-Time Compiled Java(HotSpot), C#(.NET), JavaScript(V8) 実行時に機械語に変換
高速化と柔軟性の両立
トランスパイル型 Transpiled Language TypeScript→JavaScript, Sass→CSS 他の言語に変換
新機能を古い環境で実行

JIT（Just-In-Time）コンパイル

JITコンパイルの特徴

  • 実行時に頻繁に使用される部分（ホットスポット）を検出
  • 該当部分を機械語にコンパイル
  • 次回以降は高速な機械語を実行
  • 実行プロファイルに基づく最適化

用途別の選択指針

用途 推奨される実行方式 理由
システムプログラミング コンパイル型 高速実行・メモリ効率・ハードウェア制御
Web開発 インタプリタ型/JIT型 開発効率・動的機能・迅速なデプロイ
データ分析・AI インタプリタ型 対話的実行・豊富なライブラリ・試行錯誤
モバイルアプリ コンパイル型/JIT型 パフォーマンス・バッテリー効率
ゲーム開発 コンパイル型 最高性能・リアルタイム処理
エンタープライズ バイトコード型 プラットフォーム独立・管理性・安定性

  • パフォーマンス重視：コンパイル型（C/C++/Rust/Go）
