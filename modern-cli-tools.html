モダンCLIツール vs UNIX系コマンド比較表

伝統的なUNIX系コマンドと、それらを改良・拡張した現代的なツールの比較です。

テキスト検索 grep ripgrep (rg) 代替 超高速、デフォルトで再帰検索、.gitignore対応

ファイル操作ツール

fd - 高速ファイル検索

findコマンドの現代的な代替。シンプルで高速、デフォルトで.gitignore対応。

# macOS: brew install fd
# Ubuntu/Debian: sudo apt install fd-find
# Cargo: cargo install fd-find
# 基本的な使い方 fd pattern # カレントディレクトリ以下でpatternを検索 fd -e rs # .rs拡張子のファイルを検索 fd -H pattern # 隠しファイルも含めて検索 fd -E '*.log' pattern # *.logを除外して検索 fd -t f pattern # ファイルのみ検索（ディレクトリを除外） fd -t d pattern # ディレクトリのみ検索

ripgrep (rg) - 超高速テキスト検索

grepの高速な代替。コード検索に最適化されており、デフォルトで.gitignoreを尊重。

# macOS: brew install ripgrep
# Ubuntu/Debian: sudo apt install ripgrep
# Cargo: cargo install ripgrep
# 基本的な使い方 rg pattern # カレントディレクトリ以下でpatternを検索 rg -i pattern # 大文字小文字を無視 rg -w pattern # 単語単位で検索 rg -C 3 pattern # マッチの前後3行を表示 rg -t py pattern # Pythonファイルのみ検索 rg -g '*.js' pattern # globパターンでファイルを指定 rg --stats pattern # 検索統計を表示

exa / lsd - モダンなlsコマンド

lsコマンドの拡張版。カラー表示、アイコン、Git統合などの機能を提供。

# exa - macOS: brew install exa
# lsd - macOS: brew install lsd
# Ubuntu/Debian: sudo apt install exa または lsd
# exa の使い方 exa -la # 詳細表示（隠しファイル含む） exa -T # ツリー表示 exa -la --git # Git情報も表示 exa -la --icons # アイコン付き表示 exa -la --header # ヘッダー付き表示 exa -la --time-style=long-iso # ISO形式の時刻表示

bat - シンタックスハイライト付きcat

catコマンドの拡張版。シンタックスハイライト、行番号、Git差分統合を提供。

# macOS: brew install bat
# Ubuntu/Debian: sudo apt install bat
# Cargo: cargo install bat
# 基本的な使い方 bat file.rs # シンタックスハイライト付きで表示 bat -n file.rs # 行番号付き bat -A file.rs # 非表示文字も表示 bat --style=numbers file.rs # 行番号のみ表示 bat --diff file.rs # Git差分を強調表示 bat *.rs # 複数ファイルを連続表示

テキスト処理ツール

jq - JSONプロセッサ

コマンドラインでJSONデータを処理・変換するための強力なツール。

# macOS: brew install jq
# Ubuntu/Debian: sudo apt install jq
# バイナリダウンロード: https://stedolan.github.io/jq/
# 基本的な使い方 cat data.json | jq '.' # 整形して表示 cat data.json | jq '.items[]' # items配列の各要素を表示 cat data.json | jq '.items[0].name' # 特定の値を取得 cat data.json | jq '.items | length' # 配列の長さを取得 cat data.json | jq '.items[] | select(.price > 100)' # フィルタリング cat data.json | jq '.items[] | {name, price}' # 特定のフィールドのみ抽出

yq - YAMLプロセッサ

jqのYAML版。YAMLファイルの処理・変換に特化。

# macOS: brew install yq
# Go: go install github.com/mikefarah/yq/v4@latest
# バイナリダウンロード: https://github.com/mikefarah/yq
# 基本的な使い方 yq eval '.version' config.yaml # 特定の値を取得 yq eval '.services.*.port' docker-compose.yaml # ワイルドカード使用 yq eval '.version = "2.0"' -i config.yaml # ファイルを直接編集 yq eval '. | to_json' config.yaml # JSONに変換 yq eval-all 'select(.kind == "Service")' k8s.yaml # 複数ドキュメントから選択

sd - 直感的なsed代替

sedの現代的な代替。より直感的な構文で文字列置換を実行。

# macOS: brew install sd
# Cargo: cargo install sd
# バイナリダウンロード: https://github.com/chmln/sd
# 基本的な使い方 sd 'old' 'new' file.txt # 文字列置換 sd -i 'old' 'new' file.txt # ファイルを直接編集 sd '\s+' ' ' file.txt # 正規表現で置換 sd -s 'OLD' 'new' file.txt # 大文字小文字を区別 echo "hello" | sd 'l+' 'L' # パイプでの使用

システム情報・監視ツール

htop / btop - インタラクティブプロセスビューア

topコマンドの高機能版。カラフルなUI、マウス操作、詳細な情報表示。

# htop - macOS: brew install htop
# btop - macOS: brew install btop
# Ubuntu/Debian: sudo apt install htop または btop
  • CPU、メモリ、ネットワークの使用状況をグラフ表示
  • プロセスツリー表示
  • マウスでの操作サポート
  • プロセスのフィルタリング・検索
  • カスタマイズ可能なレイアウト

dust - 視覚的なディスク使用量表示

duコマンドの現代的な代替。ディスク使用量を視覚的に表示。

# macOS: brew install dust
# Cargo: cargo install du-dust
# バイナリダウンロード: https://github.com/bootandy/dust
# 基本的な使い方 dust # カレントディレクトリの使用量表示 dust -n 10 # 上位10項目のみ表示 dust -d 3 # 深さ3まで表示 dust -r # 逆順（小さい順）で表示 dust -b # バイト単位で表示 dust ~/Documents # 特定のディレクトリを分析

duf - モダンなdfユーティリティ

dfコマンドの改良版。ディスク使用状況を見やすく表示。

# macOS: brew install duf
# Ubuntu/Debian: sudo apt install duf
# Go: go install github.com/muesli/duf@latest
# 基本的な使い方 duf # すべてのマウントポイントを表示 duf /home # 特定のパスの情報を表示 duf --only local # ローカルファイルシステムのみ duf --sort size # サイズでソート duf --json # JSON形式で出力

ネットワークツール

httpie - 人間のためのHTTPクライアント

curlの使いやすい代替。直感的な構文でHTTPリクエストを送信。

# macOS: brew install httpie
# Ubuntu/Debian: sudo apt install httpie
# pip: pip install httpie
# 基本的な使い方 http GET httpbin.org/get # GETリクエスト http POST httpbin.org/post name=John age=30 # POSTリクエスト（JSON） http -f POST httpbin.org/post name=John # フォームデータとして送信 http PUT httpbin.org/put < data.json # ファイルからデータを送信 http --download example.com/file.zip # ファイルをダウンロード http --auth user:pass example.com # 基本認証

mtr - ネットワーク診断ツール

pingとtracerouteを組み合わせた高機能ネットワーク診断ツール。

# macOS: brew install mtr
# Ubuntu/Debian: sudo apt install mtr
# 注: 一部のシステムではroot権限が必要
# 基本的な使い方 mtr google.com # インタラクティブモード mtr -r -c 10 google.com # レポートモード（10回のping） mtr -n google.com # IPアドレスのみ表示（DNS解決しない） mtr -b google.com # ホスト名とIPアドレス両方表示 mtr -4 google.com # IPv4のみ使用

bandwhich - 帯域幅モニター

どのプロセスがネットワーク帯域を使用しているかをリアルタイムで表示。

# macOS: brew install bandwhich
# Cargo: cargo install bandwhich
# 注: root権限が必要
  • プロセスごとの帯域使用量表示
  • 接続先のIPアドレス・ホスト名表示
  • プロトコル別の統計
  • リアルタイム更新

開発ツール

delta - Git差分の美しい表示

Git diffの出力を見やすく表示。シンタックスハイライトとサイドバイサイド表示。

# macOS: brew install git-delta
# Cargo: cargo install git-delta
# Gitの設定: git config --global core.pager delta
# Git設定（.gitconfig） [core] pager = delta [interactive] diffFilter = delta --color-only [delta] navigate = true side-by-side = true line-numbers = true syntax-theme = Dracula

lazygit - ターミナルGit UI

Gitの操作をターミナル上のGUIで実行。高速で直感的。

# macOS: brew install lazygit
# Ubuntu/Debian: sudo add-apt-repository ppa:lazygit-team/release && sudo apt install lazygit
# Go: go install github.com/jesseduffield/lazygit@latest
  • ステージング・コミットの視覚的操作
  • ブランチ管理・マージ
  • 対話的なリベース
  • stashの管理
  • 差分の部分的なステージング

gh - GitHub CLI

GitHubの操作をコマンドラインから実行。PR、Issue、Actionsなどを管理。

# macOS: brew install gh
# Ubuntu/Debian: https://github.com/cli/cli/blob/trunk/docs/install_linux.md
# 認証: gh auth login
# 基本的な使い方 gh repo create my-project --public # リポジトリ作成 gh pr create --title "Fix bug" --body "..." # PR作成 gh pr list # PR一覧 gh issue create --title "Bug report" # Issue作成 gh workflow run tests.yml # GitHub Actions実行 gh repo clone owner/repo # リポジトリをクローン

fzf - ファジーファインダー

あらゆるリストから対話的に項目を選択。シェル統合で作業効率が大幅向上。

# macOS: brew install fzf
# Ubuntu/Debian: sudo apt install fzf
# インストール後: $(brew --prefix)/opt/fzf/install
# 基本的な使い方 find . | fzf # ファイル選択 history | fzf # コマンド履歴から選択 git branch | fzf # ブランチ選択 # シェル統合（.bashrc/.zshrcに追加） # Ctrl-R: コマンド履歴検索 # Ctrl-T: ファイル検索 # Alt-C: ディレクトリ移動

z / zoxide - スマートディレクトリジャンプ

頻繁に使用するディレクトリに素早く移動。使用頻度を学習して賢くなる。

# zoxide (推奨) - macOS: brew install zoxide
# Ubuntu/Debian: curl -sS https://raw.githubusercontent.com/ajeetdsouza/zoxide/main/install.sh | bash
# シェル設定: eval "$(zoxide init bash)" # または zsh
# 基本的な使い方（zoxide） z proj # "proj"を含むディレクトリへ移動 z foo bar # "foo"と"bar"を含むディレクトリへ zi # インタラクティブ選択 z - # 前のディレクトリへ z .. # 親ディレクトリへ # 学習させる cd ~/projects/my-app # 通常のcdでディレクトリ移動 z my-app # 次回から簡単にジャンプ可能

その他の便利なツール

tldr - 実用的なmanページ

コマンドの使用例を簡潔に表示。manページより実用的。

# macOS: brew install tldr
# npm: npm install -g tldr
# pip: pip install tldr
tldr tar # tarコマンドの使用例 tldr git commit # git commitの使用例 tldr --update # ページキャッシュを更新

direnv - 環境変数の自動管理

ディレクトリごとに環境変数を自動的に設定・解除。

# macOS: brew install direnv
# Ubuntu/Debian: sudo apt install direnv
# シェル設定が必要: https://direnv.net/docs/hook.html
# .envrcファイルを作成 echo 'export API_KEY="secret"' > .envrc direnv allow # 信頼する cd .. # 環境変数が自動的に解除される cd - # 環境変数が自動的に設定される

asdf - 複数言語バージョン管理

Node.js、Python、Ruby等の複数言語のバージョンを統一的に管理。

# macOS: brew install asdf
# Git: git clone https://github.com/asdf-vm/asdf.git ~/.asdf
# シェル設定が必要
# プラグインインストール asdf plugin add nodejs asdf plugin add python # バージョンインストール・設定 asdf install nodejs 18.17.0 asdf global nodejs 18.17.0 asdf local python 3.11.4 # プロジェクトごとの設定

導入のヒント

  • まずは1-2個のツールから始めて、徐々に増やしていく
  • エイリアスを設定して従来のコマンドを置き換える（例: alias ls='exa'）
  • 各ツールの設定ファイルをdotfilesリポジトリで管理する
  • チームで使う場合は、READMEに必要なツールのインストール方法を記載
  • パフォーマンスが重要な場面では従来のコマンドも検討する
