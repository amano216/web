コンピューター概念一覧

コンピューターサイエンスの重要な概念と用語をまとめたリファレンスです。基本的なデータ表現から、アーキテクチャ、ネットワーク、セキュリティまで、コンピューターを理解するために必要な概念を網羅しています。

データ表現の基本

ビットとバイト

単位 英語名 (English) 説明・定義
ビット Bit (Binary Digit) コンピューターの最小単位。0または1の値を持つ。
電気信号のON/OFFで表現される。
バイト Byte 8ビットで構成される単位。
1文字を表現できる基本単位（ASCII）。
キロバイト (KB) Kilobyte 1,024バイト（2¹⁰バイト）
※SI単位系では1,000バイト
メガバイト (MB) Megabyte 1,024キロバイト（2²⁰バイト）
※SI単位系では1,000,000バイト
ギガバイト (GB) Gigabyte 1,024メガバイト（2³⁰バイト）
※SI単位系では1,000,000,000バイト
テラバイト (TB) Terabyte 1,024ギガバイト（2⁴⁰バイト）

数値表現システム

進数 英語名 基数・表記例
2進数 Binary 基数2（0, 1）
1010₂ = 10₁₀
8進数 Octal 基数8（0-7）
12₈ = 10₁₀
10進数 Decimal 基数10（0-9）
日常的に使用する数値表現
16進数 Hexadecimal 基数16（0-9, A-F）
0xFF = 255₁₀
色コードやメモリアドレスで使用

文字エンコーディング

エンコーディング 説明 特徴
ASCII American Standard Code for Information Interchange 7ビット（128文字）
英数字と基本記号のみ
Unicode Universal Character Set 世界中の文字を統一的に扱う
100万以上の文字を定義
UTF-8 8-bit Unicode Transformation Format 可変長エンコーディング（1-4バイト）
ASCIIと互換性あり
Webの標準
UTF-16 16-bit Unicode Transformation Format 2または4バイトで文字を表現
JavaやWindowsで使用
Shift_JIS Shift Japanese Industrial Standards 日本語文字エンコーディング
レガシーシステムで使用

コンピューターアーキテクチャ

主要なコンポーネント

コンポーネント 英語名 役割・機能
CPU Central Processing Unit 中央処理装置。命令の実行と計算を行う。
制御装置と演算装置から構成。
メモリ (RAM) Random Access Memory 揮発性の主記憶装置。
プログラムとデータを一時的に格納。
ストレージ Storage (HDD/SSD) 永続的なデータ保存装置。
HDD（磁気）、SSD（フラッシュメモリ）。
マザーボード Motherboard 各コンポーネントを接続する基板。
バス、スロット、コネクタを提供。
GPU Graphics Processing Unit 画像処理専用プロセッサ。
並列処理に特化、AI/MLでも活用。

メモリ階層

階層 種類 速度・容量・特徴
レジスタ CPU Register 最高速・最小容量
CPUに内蔵、即座にアクセス可能
L1キャッシュ Level 1 Cache 超高速・小容量（32-64KB）
CPU内蔵、命令とデータ分離
L2キャッシュ Level 2 Cache 高速・中容量（256KB-1MB）
CPU内蔵またはコア共有
L3キャッシュ Level 3 Cache 中速・大容量（8-32MB）
複数コアで共有
主記憶 Main Memory (RAM) 中速・大容量（8-64GB）
プログラム実行時のデータ格納
補助記憶 Secondary Storage 低速・超大容量（TB単位）
永続的なデータ保存
フォン・ノイマン型アーキテクチャ
現代のコンピューターの基本設計。プログラムとデータを同じメモリに格納し、CPUが順次実行する方式。「プログラム内蔵方式」とも呼ばれる。

オペレーティングシステム

OSの主要機能

機能 英語名 説明
プロセス管理 Process Management プログラムの実行単位を管理。
CPU時間の割り当て、スケジューリング。
メモリ管理 Memory Management 仮想メモリ、ページング、セグメンテーション。
メモリの割り当てと解放。
ファイルシステム File System データの永続的な保存と管理。
ディレクトリ構造、アクセス権限。
デバイス管理 Device Management ハードウェアデバイスの制御。
デバイスドライバ、I/O管理。
ユーザーインターフェース User Interface CLI（コマンドライン）またはGUI。
ユーザーとシステムの対話。

重要な概念

概念 英語名 説明
カーネル Kernel OSの中核部分。ハードウェアとソフトウェアの橋渡し
シェル Shell ユーザーとカーネルの仲介。コマンドの解釈と実行
システムコール System Call アプリケーションがOSの機能を利用するためのAPI
割り込み Interrupt ハードウェアやソフトウェアからのイベント通知
仮想化 Virtualization 物理リソースを論理的に分割・統合する技術

ネットワーク

OSI参照モデル

層 名称 機能・プロトコル例
第7層 アプリケーション層
(Application Layer)		 ユーザーアプリケーション
HTTP、FTP、SMTP、DNS
第6層 プレゼンテーション層
(Presentation Layer)		 データ形式の変換、暗号化
SSL/TLS、JPEG、GIF
第5層 セッション層
(Session Layer)		 通信の開始・終了管理
SQL、RPC
第4層 トランスポート層
(Transport Layer)		 信頼性のあるデータ転送
TCP、UDP
第3層 ネットワーク層
(Network Layer)		 経路選択、論理アドレス
IP、ICMP、ルーティング
第2層 データリンク層
(Data Link Layer)		 隣接ノード間の通信
Ethernet、Wi-Fi、PPP
第1層 物理層
(Physical Layer)		 ビット列の伝送
電気信号、光信号、無線

セキュリティ

暗号化技術

種類 英語名 特徴・用途
対称鍵暗号 Symmetric Encryption 同じ鍵で暗号化・復号化
高速、大量データ向き
AES、DES、3DES
非対称鍵暗号 Asymmetric Encryption 公開鍵と秘密鍵のペア
鍵配送問題を解決
RSA、楕円曲線暗号
ハッシュ関数 Hash Function 一方向性関数
データの完全性確認
SHA-256、MD5（非推奨）
デジタル署名 Digital Signature 送信者の認証と改ざん検出
秘密鍵で署名、公開鍵で検証
SSL/TLS Secure Sockets Layer/Transport Layer Security Webの暗号化通信
HTTPS実現の基盤技術

セキュリティの基本概念

概念 英語名 説明
機密性 Confidentiality 許可されたユーザーのみがアクセス可能
完全性 Integrity データが改ざんされていないことを保証
可用性 Availability 必要な時にシステムが利用可能
認証 Authentication ユーザーやシステムの身元確認
認可 Authorization リソースへのアクセス権限の付与

一般的な攻撃手法

攻撃手法 英語名 説明
SQLインジェクション SQL Injection データベースへの不正なSQL文の注入
XSS Cross-Site Scripting Webページに悪意のあるスクリプトを埋め込む
CSRF Cross-Site Request Forgery ユーザーの意図しないリクエストを強制実行
DDoS Distributed Denial of Service 大量のリクエストでサービスを停止させる
中間者攻撃 Man-in-the-Middle Attack 通信経路に割り込んで情報を盗聴・改ざん

人工知能・機械学習

AI/MLの基本概念

概念 英語名 説明
