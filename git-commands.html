Git/GitHubコマンドリファレンス

バージョン管理システムGitとGitHub CLIの主要なコマンドをまとめました。

基本的なGitコマンド

初期設定・設定確認

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
git init Initialize 新しいGitリポジトリを初期化する。現在のディレクトリをGitリポジトリとして初期化
git config Configure Gitの設定を行う。ユーザー名やメールアドレスなどを設定
git config --list List Configuration 現在のGit設定を一覧表示する。グローバル設定とローカル設定を確認
git clone Clone リモートリポジトリを複製する。GitHubなどから完全なコピーを作成

基本操作

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
git status Status 作業ツリーの状態を表示する。変更されたファイルやステージング状態を確認
git add Add ファイルをステージングエリアに追加する。コミット前の準備段階
git commit Commit 変更をコミットする。ステージングエリアの内容を履歴に記録
git diff Difference 変更内容の差分を表示する。作業中の変更やコミット間の違いを確認
git rm Remove ファイルを削除する。Gitの管理下からファイルを除外
git mv Move ファイルを移動または名前変更する。履歴を保持したままファイル名を変更

履歴の確認

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
git log Log コミット履歴を表示する。過去の変更履歴を時系列で確認
git show Show コミットの詳細を表示する。特定のコミットの変更内容を確認
git blame Blame 各行の最終変更者を表示する。誰がいつ変更したかを行単位で確認
git reflog Reference Log 参照ログを表示する。HEADの移動履歴を確認して操作を復元可能

ブランチ操作

ブランチの作成・切り替え

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
git branch Branch ブランチの一覧表示または作成する。分岐した開発ラインを管理
git checkout Check Out ブランチを切り替える。別のブランチやコミットに移動
git switch Switch ブランチを切り替える（新しい方法）。checkoutより明確なブランチ切り替え
git merge Merge ブランチをマージする。分岐した変更を統合
git branch -d Delete Branch ブランチを削除する。マージ済みブランチを安全に削除

リモートリポジトリ操作

リモートの管理

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
git remote Remote リモートリポジトリを管理する。GitHubなどのリモート接続設定
git fetch Fetch リモートの変更を取得する。ローカルにダウンロードするがマージしない
git pull Pull リモートの変更を取得してマージする。fetchとmergeを同時実行
git push Push 変更をリモートに送信する。ローカルのコミットをリモートにアップロード

変更の取り消し

コミットやステージングの取り消し

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
git reset Reset コミットを取り消す。作業内容を過去の状態に戻す
git revert Revert コミットを打ち消す新しいコミットを作成。履歴を保持したまま変更を元に戻す
git restore Restore 作業ツリーのファイルを復元する。変更を破棄して元の状態に戻す
git clean Clean 追跡されていないファイルを削除する。作業ディレクトリをクリーンアップ

高度な操作

リベースとチェリーピック

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
git rebase Rebase コミット履歴を整理する。ブランチの基点を変更して履歴をクリーンに保つ
git cherry-pick Cherry Pick 特定のコミットを適用する。他のブランチから必要なコミットだけを取り込む

一時保存とタグ

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
git stash Stash 変更を一時保存する。作業中の変更を退避して別の作業を行う
git tag Tag タグを作成・管理する。リリースバージョンなどの目印を付ける

GitHub CLI (gh) コマンド

リポジトリ操作

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
gh repo create Create Repository 新しいGitHubリポジトリを作成する。コマンドラインから直接作成
gh repo clone Clone Repository GitHubリポジトリをクローンする。簡単な書式でクローン可能
gh repo view View Repository リポジトリ情報を表示する。READMEやステータスを確認

プルリクエスト操作

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
gh pr create Create Pull Request プルリクエストを作成する。コマンドラインからPRを作成
gh pr list List Pull Requests プルリクエスト一覧を表示する。オープン中のPRを確認
gh pr view View Pull Request プルリクエストの詳細を表示する。コメントやレビューを確認
gh pr checkout Check Out Pull Request PRをローカルにチェックアウトする。レビューやテスト用にローカルで確認
gh pr merge Merge Pull Request プルリクエストをマージする。コマンドラインから直接マージ

Issue操作

コマンド 英語の意味 (Meaning) 日本語の意味・使用方法
gh issue create Create Issue 新しいIssueを作成する。バグ報告や機能要望を記録
gh issue list List Issues Issue一覧を表示する。オープン中の課題を確認
gh issue view View Issue Issueの詳細を表示する。問題の内容やコメントを確認
gh issue close Close Issue Issueをクローズする。解決済みの問題を終了

よく使うGitエイリアス設定

便利なエイリアスの例

エイリアス 実際のコマンド (Actual Command) 設定方法・説明
git st git status statusコマンドの省略形。頻繁に使うコマンドを短く
git co git checkout checkoutコマンドの省略形。ブランチ切り替えを素早く
git br git branch branchコマンドの省略形。ブランチ一覧を簡単に
git cm git commit -m コミットメッセージ付きコミットの省略形
git lg Formatted Log Display グラフ表示付きの見やすいログ。色付きで履歴を視覚化

トラブルシューティング

よくある問題と解決方法

問題 解決方法 (Solution) 説明・対処法
間違ってコミットした Amend Commit git commit --amendで直前のコミットを修正。メッセージや内容を変更
マージコンフリクト Resolve Conflicts git statusで確認後、手動修正。コンフリクトマーカーを削除してadd
間違ったブランチで作業 Stash and Switch git stash → git checkout 正しいブランチ → git stash popで変更を移動
リモートと同期したい Force Sync with Remote git fetch --all → git reset --hard origin/mainでローカルをリモートに完全一致
特定のファイルだけ元に戻す Discard File Changes git checkout -- file.txtで作業中の変更を破棄。最新コミットの状態に戻す
