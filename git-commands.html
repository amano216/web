Git/GitHubコマンドリファレンス

バージョン管理システムGitとGitHub CLIの主要なコマンドをまとめました。

基本的なGitコマンド

初期設定・設定確認

コマンド 説明 使用例
git init 新しいGitリポジトリを初期化
git init
git config Gitの設定を行う
git config --global user.name "名前"
git config --global user.email "email@example.com"
git config --list 現在の設定を一覧表示
git config --list
git clone リモートリポジトリをクローン
git clone https://github.com/user/repo.git

基本操作

コマンド 説明 使用例
git status 作業ツリーの状態を表示
git status
git add ファイルをステージングエリアに追加
git add file.txt
git add .
git add -A
git commit 変更をコミット
git commit -m "コミットメッセージ"
git commit -am "ステージングとコミット"
git diff 変更内容を表示
git diff
git diff --staged
git diff HEAD
git rm ファイルを削除
git rm file.txt
git rm --cached file.txt
git mv ファイルを移動/名前変更
git mv old.txt new.txt

よく使うGitエイリアス設定

便利なエイリアスの例

エイリアス 実際のコマンド 設定方法
git st git status
git config --global alias.st status
git co git checkout
git config --global alias.co checkout
git br git branch
git config --global alias.br branch
git cm git commit -m
git config --global alias.cm "commit -m"
git lg 見やすいログ表示
git config --global alias.lg "log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"
git undo 直前のコミットを取り消し
git config --global alias.undo "reset --soft HEAD^"

トラブルシューティング

よくある問題と解決方法

問題 解決コマンド 説明
間違ってコミットした git commit --amend 直前のコミットを修正
マージコンフリクト git statusで確認後、手動修正 コンフリクトファイルを編集してからgit add
間違ったブランチで作業 git stash → git checkout 正しいブランチ → git stash pop 変更を一時保存して移動
リモートと同期したい git fetch --all → git reset --hard origin/main ローカルの変更を破棄してリモートに合わせる
特定のファイルだけ元に戻す git checkout -- file.txt 作業中の変更を破棄
