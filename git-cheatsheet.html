Git/GitHub記法チートシート

Gitの基本的なワークフローと、GitHubでよく使う記法をまとめたクイックリファレンスです。

Gitの基本ワークフロー

リポジトリの初期化と設定

# 新規リポジトリの作成
git init

# リモートリポジトリのクローン
git clone https://github.com/user/repo.git
git clone git@github.com:user/repo.git  # SSH使用

# ユーザー情報の設定
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 設定の確認
git config --list
git config user.name

基本的な変更の流れ

# 状態の確認
git status
git status -s  # 短縮表示

# ファイルをステージングエリアに追加
git add file.txt
git add *.js           # ワイルドカード使用
git add .             # すべての変更を追加
git add -p            # 対話的に部分追加

# コミット
git commit -m "コミットメッセージ"
git commit -am "追加とコミットを同時に"  # 追跡済みファイルのみ
git commit --amend    # 直前のコミットを修正

# 変更内容の確認
git diff               # ワーキングツリーの変更
git diff --staged     # ステージングエリアの変更
git diff HEAD         # 最新コミットとの差分

ブランチ操作

ブランチの作成と切り替え

# ブランチ一覧
git branch             # ローカルブランチ
git branch -r          # リモートブランチ
git branch -a          # すべてのブランチ

# ブランチの作成
git branch feature/new-feature

# ブランチの切り替え
git checkout feature/new-feature
git switch feature/new-feature  # 新しい方法

# ブランチ作成と切り替えを同時に
git checkout -b feature/new-feature
git switch -c feature/new-feature

# ブランチの削除
git branch -d feature/old-feature  # マージ済み
git branch -D feature/old-feature  # 強制削除

マージとリベース

# マージ（マージコミットあり）
git checkout main
git merge feature/new-feature

# ファストフォワードマージ
git merge --ff-only feature/new-feature

# マージの中止
git merge --abort

# リベース
git checkout feature/new-feature
git rebase main

# インタラクティブリベース（コミット履歴の整理）
git rebase -i HEAD~3  # 直近3コミットを編集

リモートリポジトリとの連携

リモート操作

# リモートリポジトリの確認
git remote -v

# リモートリポジトリの追加
git remote add origin https://github.com/user/repo.git

# リモートの変更を取得（マージなし）
git fetch
git fetch origin

# プル（fetch + merge）
git pull
git pull origin main
git pull --rebase  # リベースでプル

# プッシュ
git push
git push origin main
git push -u origin feature/new-feature  # 上流ブランチを設定
git push --force  # 強制プッシュ（危険！）

履歴の確認

ログの表示

# 基本的なログ表示
git log
git log --oneline      # 1行表示
git log --graph        # グラフ表示
git log --oneline --graph --all  # 全ブランチをグラフで

# フィルタリング
git log -n 5          # 直近5件
git log --since="2 weeks ago"
git log --author="John"
git log --grep="bug fix"

# ファイル単位の履歴
git log --follow file.txt
git blame file.txt   # 行ごとの最終変更者

変更の取り消し

様々な取り消し方法

# ステージングの取り消し
git reset file.txt
git restore --staged file.txt  # 新しい方法

# ワーキングツリーの変更を取り消し
git checkout -- file.txt
git restore file.txt  # 新しい方法

# コミットの取り消し（履歴は残る）
git revert HEAD      # 直前のコミットを打ち消す
git revert abc123    # 特定のコミットを打ち消す

# コミットのリセット（履歴から削除）
git reset --soft HEAD~1   # コミットのみ取り消し
git reset --mixed HEAD~1  # コミットとステージング取り消し
git reset --hard HEAD~1   # すべて取り消し（危険！）

スタッシュ（一時保存）

作業内容の一時保存

# スタッシュに保存
git stash
git stash save "作業中の機能"
git stash -u           # 未追跡ファイルも含む

# スタッシュ一覧
git stash list

# スタッシュの適用
git stash pop          # 適用して削除
git stash apply        # 適用のみ（削除しない）
git stash apply stash@{1}  # 特定のスタッシュ

# スタッシュの削除
git stash drop
git stash clear        # すべて削除

タグ管理

バージョンタグの作成

# タグ一覧
git tag
git tag -l "v1.*"     # パターンマッチ

# 軽量タグ
git tag v1.0.0

# 注釈付きタグ（推奨）
git tag -a v1.0.0 -m "バージョン1.0.0リリース"

# 過去のコミットにタグ付け
git tag -a v0.9.0 abc123

# タグをリモートにプッシュ
git push origin v1.0.0
git push origin --tags  # すべてのタグ

# タグの削除
git tag -d v1.0.0
git push origin --delete v1.0.0

GitHub特有の記法

Markdown記法（README.md、Issue、PR）

記法 英語の意味 (Meaning) 表示結果・用途
# 見出し1 Heading 1 最大の見出し
## 見出し2 Heading 2 セクション見出し
**太字** Bold 太字テキスト
*斜体* Italic 斜体テキスト
~~取り消し線~~ Strikethrough 取り消し線
`コード` Inline Code インラインコード
```言語名 Code Block コードブロック（シンタックスハイライト付き）
[リンク](URL) Link ハイパーリンク
![代替テキスト](画像URL) Image 画像の埋め込み
- [ ] タスク Task List チェックボックス付きタスクリスト

GitHub特有の参照記法

記法 英語の意味 (Meaning) 用途・説明
#123 Issue Reference Issue番号123を参照
@username Mention ユーザーにメンション（通知される）
user/repo#123 Cross-repo Reference 他のリポジトリのIssueを参照
abc123 Commit SHA コミットへのリンク（7文字以上）
:emoji: Emoji 絵文字（:smile:、:+1:など）

Fixes #123, fixes #124

.gitignoreの記法

ファイル無視パターン

# コメント

# 特定のファイル
secret.txt

# ワイルドカード
*.log          # すべての.logファイル
*.tmp          # すべての.tmpファイル

# ディレクトリ
node_modules/  # node_modulesディレクトリ全体
dist/          # distディレクトリ全体

# 特定の場所のファイル
/config.json   # ルートのconfig.jsonのみ
src/*.test.js  # srcディレクトリ直下のテストファイル

# 除外の例外
*.log
!important.log # important.logは追跡する

# 再帰的なパターン
**/*.bak       # すべてのディレクトリの.bakファイル

便利なGitエイリアス

よく使うエイリアスの設定

# エイリアスの設定
git config --global alias.co 'checkout'
git config --global alias.br 'branch'
git config --global alias.ci 'commit'
git config --global alias.st 'status'
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'
git config --global alias.lg 'log --oneline --graph --all'

# 使用例
git co main
git st
git lg
