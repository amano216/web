HTMLタグ一覧表

Webページを作成するために使用される主要なHTMLタグを、機能ごとにまとめました。

基本的な文書構造

すべてのHTML文書の基本となる骨格を定義します。

タグ 英語の意味 (Meaning) 日本語の意味・用途
<!DOCTYPE html> Document Type Declaration この文書がHTML5で書かれていることを宣言します。
<html> HTML Document HTML文書全体のルート（根元）要素です。
<head> Document Head 文書のメタデータ（タイトル、文字コード、CSS、スクリプトなど）を格納します。
<body> Document Body 実際にブラウザに表示されるすべてのコンテンツ（テキスト、画像、リンクなど）を格納します。

メタデータ

文書に関する情報や、他のリソースとの関連を定義します。<head>タグ内に記述します。

タグ 英語の意味 (Meaning) 日本語の意味・用途
<title> Document Title ブラウザのタブや検索結果に表示されるページのタイトルを定義します。
<meta> Metadata 文字コード、ページの概要、キーワードなど、他のメタタグで表現できない情報を指定します。
<link> Resource Link 外部のスタイルシート（CSS）やファビコンなどを文書に関連付けます。
<style> Style Information HTML文書内に直接CSSを記述するために使用します。
<script> Script JavaScriptのコードを埋め込んだり、外部のスクリプトファイルを読み込んだりします。

セクショニング

文書の構造を意味的に区切り、アウトラインを形成します。

タグ 英語の意味 (Meaning) 日本語の意味・用途
<header> Header ページやセクションのヘッダー（導入部分）を表します。サイトのロゴやナビゲーションが含まれることが多いです。
<footer> Footer ページやセクションのフッター（末尾部分）を表します。著作権情報や連絡先などが含まれます。
<main> Main Content ページの主要なコンテンツを表します。このタグは文書内に一つだけ配置します。
<nav> Navigation サイト内の主要なナビゲーションリンク（メニュー）のセクションを表します。
<article> Article 自己完結した独立したコンテンツ（ブログ投稿、ニュース記事など）を表します。
<section> Section 文書内のテーマごとのグループを表します。見出しを持つのが一般的です。
<aside> Aside 本文の補足情報や広告など、主要なコンテンツとは関連が薄いセクションを表します。
<h1>〜<h6> Heading 1-6 見出しを表します。<h1>が最上位の見出しで、<h6>が最も低いレベルの見出しです。
<div> Division 特定の意味を持たない汎用的なブロックレベルのコンテナです。CSSでのスタイリングやJSでの操作のために使われます。

テキストコンテンツ

段落やリスト、引用など、テキストの構造や意味を定義します。

タグ 英語の意味 (Meaning) 日本語の意味・用途
<p> Paragraph 段落を表します。
<a> Anchor 他のページやファイル、ページ内の特定箇所へのハイパーリンクを作成します。
<span> Span 特定の意味を持たない汎用的なインラインコンテナです。テキストの一部にスタイルを適用する場合などに使います。
<strong> Strong Importance 内容の重要性が高いことを示します。通常は太字で表示されます。
<em> Emphasis テキストを強調（emphase）することを示します。通常は斜体で表示されます。
<b> Bring Attention To 他のテキストと区別したい部分に使います。重要性や強調の意味合いはありません。通常は太字で表示されます。（例：製品名）
<i> Idiomatic Text 声や気分が違う部分に使います。専門用語や外国語など。通常は斜体で表示されます。
<br> Line Break 改行を強制します。
<hr> Thematic Break 段落レベルでのテーマの区切り（横罫線）を表します。
<blockquote> Blockquote 他のソースからの引用ブロックを表します。通常はインデントされます。
<ul> Unordered List 順序のないリストを作成します。
<ol> Ordered List 順序のあるリスト（番号付きリスト）を作成します。
<li> List Item リストの各項目を表します。<ul>または<ol>の中で使います。
<code> Code コンピュータのコードの一部を表します。
<pre> Preformatted Text 整形済みのテキストを表します。スペースや改行がそのまま表示されます。

埋め込みコンテンツ

画像や動画など、外部のメディアをページに埋め込みます。

タグ 英語の意味 (Meaning) 日本語の意味・用途
<img> Image 画像を表示します。src属性で画像のパスを、alt属性で代替テキストを指定します。
<video> Video 動画を埋め込みます。
<audio> Audio 音声ファイルを埋め込みます。
<iframe> Inline Frame 他のHTMLページを現在のページに埋め込みます。

テーブル

表形式のデータを表現します。

タグ 英語の意味 (Meaning) 日本語の意味・用途
<table> Table 表全体を囲みます。
<thead> Table Head 表のヘッダー行のグループを定義します。
<tbody> Table Body 表の本体（データ行）のグループを定義します。
<tfoot> Table Foot 表のフッター行のグループを定義します。
<tr> Table Row 表の行を定義します。
<th> Table Header 表の見出しセルを定義します。
<td> Table Data 表のデータセルを定義します。
<caption> Caption 表のキャプション（タイトルや説明）を定義します。

フォーム

ユーザーがデータを入力し、サーバーに送信するためのコントロールを作成します。

タグ 英語の意味 (Meaning) 日本語の意味・用途
<form> Form ユーザー入力のためのフォーム全体を定義します。
<input> Input Control テキストフィールド、チェックボックス、ラジオボタン、送信ボタンなど、様々な入力欄を作成します。
<textarea> Text Area 複数行のテキスト入力欄を作成します。
<button> Button クリック可能なボタンを作成します。
<label> Label フォームの入力コントロール (<input>など) のためのラベル（見出し）を定義します。
<select> Select Box ドロップダウンリストを作成します。
<option> Option <select>タグ内で使用し、ドロップダウンリストの各項目を定義します。
<fieldset> Field Set フォーム内の関連するコントロールをグループ化します。
<legend> Legend <fieldset>でグループ化された部分にキャプション（見出し）を付けます。

<img>タグの主な属性一覧

画像を表示する<img>タグで使用できる主要な属性をまとめました。

属性名 説明 使用例
src 画像ファイルのURLやパスを指定します（必須属性）。 <img src="images/photo.jpg">
alt 画像が表示できない場合の代替テキスト。アクセシビリティのため必須。 <img src="..." alt="富士山の写真">
width 画像の幅をピクセル単位で指定します。 <img src="..." width="300">
height 画像の高さをピクセル単位で指定します。 <img src="..." height="200">
loading 画像の読み込みタイミングを制御。"lazy"で遅延読み込みを有効化。 <img src="..." loading="lazy">
srcset 異なる解像度の画像を複数指定し、デバイスに応じて最適な画像を選択。 <img src="..." srcset="small.jpg 300w, large.jpg 1000w">
sizes レスポンシブ画像で、画像の表示サイズを指定。srcsetと併用。 <img srcset="..." sizes="(max-width: 600px) 100vw, 50vw">
title 画像にマウスを合わせた時に表示されるツールチップテキスト。 <img src="..." title="クリックで拡大">
class CSSクラスを指定してスタイルを適用。 <img src="..." class="thumbnail rounded">
id 要素に一意のIDを付与。JavaScriptやCSSでの操作に使用。 <img src="..." id="main-image">
style インラインCSSスタイルを直接指定。 <img src="..." style="border-radius: 10px;">
decoding 画像のデコード方法を指定。"async"で非同期デコードを有効化。 <img src="..." decoding="async">
crossorigin CORSの設定。"anonymous"や"use-credentials"を指定。 <img src="..." crossorigin="anonymous">
usemap クライアントサイドイメージマップを使用する場合のmap要素への参照。 <img src="..." usemap="#map1">

<a>タグの主な属性一覧

属性名 説明 使用例

class 要素にクラス名を付与し、CSSでまとめてスタイルを適用します。 <a href="..." class="button">ボタン</a>

セマンティックHTML（意味論的HTML）

セマンティックHTMLとは

HTMLタグを、その内容の意味や目的に応じて適切に使用することです。見た目ではなく、構造と意味を重視します。

概念 説明 メリット
意味の明確化 タグ自体が内容の意味を表現する コードの可読性向上、開発者間の理解促進
アクセシビリティ スクリーンリーダーなどの支援技術が内容を正確に解釈 視覚障害者を含むすべてのユーザーが利用可能
SEO効果 検索エンジンがページ構造を正確に理解 検索順位の向上、適切なインデックス化
保守性 構造が明確なため、後からの修正が容易 長期的なメンテナンスコストの削減

セマンティックタグの使い分け

用途 セマンティック（推奨） 非セマンティック（避ける） 理由
ナビゲーション <nav> <div class="navigation"> navタグは明確にナビゲーションであることを示す
記事・投稿 <article> <div class="post"> 独立したコンテンツであることを明示
補足情報 <aside> <div class="sidebar"> 主要コンテンツとの関係性を明確化
日付・時刻 <time datetime="2024-01-01"> <span>2024年1月1日</span> 機械が読み取り可能な形式で時刻を表現
連絡先情報 <address> <div class="contact"> 連絡先情報であることを明示的に表現

WAI-ARIA属性の基本

WAI-ARIA（Web Accessibility Initiative - Accessible Rich Internet Applications）は、Webコンテンツのアクセシビリティを向上させるための属性です。

属性 説明 使用例
role 要素の役割を明示的に定義 <div role="navigation">
aria-label 要素にアクセシブルな名前を提供 <button aria-label="閉じる">×</button>
aria-labelledby 他の要素のIDを参照してラベルを設定 <div aria-labelledby="heading1">
aria-describedby 要素の詳細な説明を提供する要素を参照 <input aria-describedby="help-text">
aria-hidden 装飾的な要素を支援技術から隠す <span aria-hidden="true">🎨</span>
aria-live 動的に変更される領域を通知 <div aria-live="polite">更新通知</div>
aria-required フォーム入力が必須であることを示す <input aria-required="true">
aria-invalid 入力値が無効であることを示す <input aria-invalid="true">
aria-expanded 要素の展開/折りたたみ状態を示す <button aria-expanded="false">メニュー</button>
aria-current 現在の項目を示す（ナビゲーションなど） <a aria-current="page">ホーム</a>

HTMLのベストプラクティス

より良いHTML文書を作成するための推奨事項です。

カテゴリ ベストプラクティス 理由・効果
文書構造 見出しタグ（h1〜h6）を階層順に使用 文書のアウトラインが明確になり、アクセシビリティが向上
画像 すべての画像にalt属性を設定 画像が表示されない場合の代替テキスト、アクセシビリティ向上
フォーム label要素を必ず使用し、for属性でinputと関連付け クリック可能領域の拡大、アクセシビリティの向上
言語設定 html要素にlang属性を設定 スクリーンリーダーの正確な読み上げ、SEO効果
文字エンコーディング UTF-8を使用し、meta charsetで宣言 文字化けの防止、国際化対応
外部リンク target="_blank"使用時はrel="noopener"を追加 セキュリティリスクの回避（リンク先からの操作を防ぐ）
パフォーマンス 重要でない画像にloading="lazy"を使用 初期表示速度の向上、帯域幅の節約
レスポンシブ対応 viewport metaタグを設定 モバイルデバイスでの適切な表示
スクリプト配置 scriptタグは基本的にbody終了タグの直前に配置 ページ表示のブロッキングを防ぐ

よくある間違いと正しい使い方

HTMLでよく見られる誤用と、その正しい使用方法を紹介します。

よくある間違い 正しい使い方 理由
<br><br>で余白を作る CSSのmarginやpaddingを使用 brは改行のためのタグで、レイアウト調整には不適切
見出しをサイズで選ぶ
<h3>を大きさで選択		 階層構造に従って選択
サイズはCSSで調整		 見出しは文書の構造を表すもので、見た目のためではない
テーブルでレイアウト
<table>でページ構成		 CSSのFlexboxやGridを使用 tableは表形式のデータ専用、レイアウトには不適切
空のalt属性を省略
<img src="...">		 装飾画像でも空のaltを設定
<img src="..." alt="">		 alt属性は必須、装飾画像は空文字列で示す
divの過剰使用
<div><div><div>		 適切なセマンティックタグを使用 divは最終手段、まずは意味のあるタグを検討
インラインスタイルの多用
style="color:red"		 外部CSSまたはstyleタグを使用 保守性の低下、再利用性の欠如
非推奨タグの使用
<center>, <font>		 CSSで代替
text-align: center		 HTML5で廃止、見た目はCSSの責務
blockquote要素でインデント
見た目のために使用		 CSSのmargin-leftを使用 blockquoteは引用専用、インデントは副作用
