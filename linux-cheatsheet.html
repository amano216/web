Linux記法チートシート

Linuxコマンドラインの基本的な記法と使い方をまとめたクイックリファレンスです。

基本的なコマンド構文

コマンドの基本構造

構文要素 英語の意味 (Meaning) 使用方法・説明
command Command 実行するコマンド名。ls, cd, cpなど
-option Option 単一文字のオプション。-l, -aなど。組み合わせ可能-la
--long-option Long Option 長い形式のオプション。--help, --version
argument Argument コマンドに渡す引数。ファイル名やパスなど
< Input Redirection ファイルから入力を受け取る
> Output Redirection 出力をファイルに書き込む（上書き）
>> Append 出力をファイルに追記
| Pipe コマンドの出力を次のコマンドの入力に接続 
# 基本的なコマンド構造
command -options arguments

# 実例
ls -la /home/user
grep -i "pattern" file.txt
tar -czf archive.tar.gz directory/

ファイル・ディレクトリ操作

パスの指定方法

# 絶対パス（ルートから）
cd /home/user/documents

# 相対パス（現在位置から）
cd documents
cd ../              # 親ディレクトリへ
cd ../../           # 2階層上へ
cd ./subfolder      # 現在のディレクトリのsubfolderへ

# 特殊なパス
cd ~                # ホームディレクトリへ
cd -                # 前のディレクトリへ戻る

ファイル操作の基本パターン

# ファイルのコピー
cp source.txt destination.txt
cp -r source_dir/ dest_dir/    # ディレクトリを再帰的にコピー
cp -p file.txt backup.txt    # 権限を保持してコピー

# ファイルの移動・名前変更
mv old_name.txt new_name.txt
mv file.txt /path/to/directory/

# ファイルの削除
rm file.txt
rm -i *.log           # 確認しながら削除
rm -rf directory/     # ディレクトリを強制削除（危険！）

# ディレクトリの作成
mkdir new_directory
mkdir -p parent/child/grandchild  # 親ディレクトリも作成

ワイルドカード（グロブパターン）

パターンマッチング

パターン 英語の意味 (Meaning) 説明・使用例
* Asterisk 0文字以上の任意の文字。*.txtで全てのtxtファイル
? Question Mark 任意の1文字。file?.txtでfile1.txt等
[abc] Character Class 指定した文字のいずれか。[0-9]で数字
[!abc] Negation 指定以外の文字。[!0-9]で数字以外
{a,b,c} Brace Expansion 複数の選択肢。{txt,log,csv} 
# ワイルドカードの使用例
ls *.jpg                    # 全てのJPGファイル
ls IMG_????.jpg            # IMG_0001.jpg のような4桁の番号
ls [A-Z]*                  # 大文字で始まるファイル
ls *.[ch]                  # .cまたは.hファイル
rm temp_{1,2,3}.txt        # temp_1.txt, temp_2.txt, temp_3.txt
cp file.{txt,bak}           # file.txtをfile.bakにコピー

リダイレクトとパイプ

入出力の制御

# 標準出力のリダイレクト
echo "Hello" > file.txt      # ファイルに出力（上書き）
echo "World" >> file.txt     # ファイルに追記

# 標準エラー出力のリダイレクト
command 2> error.log          # エラーをファイルに
command > output.txt 2> error.log  # 出力とエラーを別々に
command > all.log 2>&1       # 出力とエラーを同じファイルに

# 標準入力のリダイレクト
wc -l < file.txt             # ファイルから入力を読み込む

# パイプの使用
ls -la | grep "\.txt$"      # txtファイルのみ表示
cat file.txt | sort | uniq   # ソートして重複削除
ps aux | grep "nginx" | wc -l # nginxプロセス数をカウント

権限とパーミッション

権限の表記と変更

権限 数値 意味
r 4 読み取り（Read）
w 2 書き込み（Write）
x 1 実行（Execute） 
# 権限の確認
ls -l file.txt
# -rw-r--r-- 1 user group 1234 Jan 1 12:00 file.txt

# 数値モードでの権限変更
chmod 755 script.sh         # rwxr-xr-x
chmod 644 document.txt      # rw-r--r--
chmod 600 private_key       # rw-------

# シンボリックモードでの権限変更
chmod +x script.sh          # 実行権限を追加
chmod u+w file.txt          # 所有者に書き込み権限
chmod go-w file.txt         # グループと他者から書き込み権限を削除
chmod a+r public.txt        # 全員に読み取り権限

# 所有者の変更
chown user:group file.txt
chown -R user:group directory/  # 再帰的に変更

プロセス管理

プロセスの操作

# プロセスの確認
ps aux                       # 全プロセスを詳細表示
ps aux | grep "process_name"
pgrep -l "nginx"            # プロセス名で検索

# プロセスの終了
kill 1234                    # PID 1234のプロセスを終了
kill -9 1234                 # 強制終了（SIGKILL）
killall "firefox"           # プロセス名で全て終了
pkill -f "pattern"          # パターンマッチで終了

# バックグラウンド実行
command &                    # バックグラウンドで実行
nohup command &              # ログアウト後も継続
jobs                         # バックグラウンドジョブ一覧
fg %1                        # ジョブ1をフォアグラウンドへ
bg %1                        # ジョブ1をバックグラウンドで再開

テキスト処理

grepによる検索

# 基本的な検索
grep "pattern" file.txt
grep -i "pattern" file.txt   # 大文字小文字を無視
grep -v "pattern" file.txt   # パターンに一致しない行
grep -n "pattern" file.txt   # 行番号付き
grep -r "pattern" directory/ # ディレクトリを再帰検索

# 正規表現の使用
grep -E "^[0-9]+" file.txt   # 数字で始まる行
grep -E "error|warning" log.txt
grep -E "\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b" # メールアドレス

sedによる置換

# 基本的な置換
sed 's/old/new/' file.txt      # 各行の最初の一致を置換
sed 's/old/new/g' file.txt     # 全ての一致を置換
sed -i 's/old/new/g' file.txt  # ファイルを直接編集

# 行の削除・挿入
sed '5d' file.txt              # 5行目を削除
sed '/pattern/d' file.txt      # パターンに一致する行を削除
sed '2i\New line' file.txt     # 2行目の前に挿入

# 複数の操作
sed -e 's/old1/new1/g' -e 's/old2/new2/g' file.txt

awkによるデータ処理

# 列の抽出
awk '{print $1}' file.txt       # 1列目を表示
awk '{print $1, $3}' file.txt   # 1列目と3列目
awk '{print $NF}' file.txt      # 最後の列

# 条件付き処理
awk '$3 > 100 {print $0}' data.txt  # 3列目が100より大きい行
awk '/pattern/ {print $2}' file.txt  # パターンに一致する行の2列目

# 計算と集計
awk '{sum += $1} END {print sum}' numbers.txt
awk '{count++} END {print count}' file.txt  # 行数カウント

# フィールドセパレータの指定
awk -F: '{print $1}' /etc/passwd  # :を区切り文字として使用

シェルスクリプトの基礎

変数と制御構造

#!/bin/bash
# シェルスクリプトの先頭行（シバン）

# 変数の定義と使用
NAME="John"
AGE=25
echo "Name: $NAME, Age: $AGE"
echo "Name: ${NAME}, Age: ${AGE}"  # より安全な記法

# コマンドの実行結果を変数に格納
CURRENT_DIR=$(pwd)
FILE_COUNT=$(ls -1 | wc -l)

# 条件分岐
if [ $AGE -gt 18 ]; then
    echo "Adult"
elif [ $AGE -eq 18 ]; then
    echo "Just 18"
else
    echo "Minor"
fi

# ループ処理
for file in *.txt; do
    echo "Processing: $file"
    cp "$file" "backup_$file"
done

# while ループ
COUNTER=0
while [ $COUNTER -lt 10 ]; do
    echo $COUNTER
    COUNTER=$((COUNTER + 1))
done

環境変数とエイリアス

環境設定

# 環境変数の設定
export PATH="$PATH:/usr/local/bin"
export EDITOR="vim"
export LANG="ja_JP.UTF-8"

# 環境変数の確認
echo $PATH
env                          # 全環境変数を表示
printenv HOME               # 特定の環境変数

# エイリアスの設定
alias ll='ls -la'
alias gs='git status'
alias ..='cd ..'
alias ...='cd ../..'

# 関数の定義
mkcd() {
    mkdir -p "$1" && cd "$1"
}

# .bashrcや.zshrcに追加して永続化
echo "alias ll='ls -la'" >> ~/.bashrc
source ~/.bashrc            # 設定を再読み込み

便利なワンライナー

実用的なコマンドの組み合わせ

# ファイル検索と処理
find . -name "*.log" -mtime +30 -delete  # 30日以上前のログを削除
find . -type f -size +100M             # 100MB以上のファイル
find . -name "*.sh" -exec chmod +x {} \;  # .shファイルに実行権限

# ディスク使用量の確認
du -sh * | sort -hr | head -10        # サイズ順トップ10
df -h | grep -vE '^Filesystem|tmpfs'    # 実際のディスクのみ表示

# プロセス監視
ps aux | sort -nrk 3,3 | head -10      # CPU使用率トップ10
ps aux | sort -nrk 4,4 | head -10      # メモリ使用率トップ10

# ログ監視
tail -f /var/log/syslog | grep --line-buffered "ERROR"
journalctl -f -u nginx                   # nginxのログをリアルタイム監視

# ネットワーク確認
ss -tuln | grep LISTEN                  # リスニングポート一覧
curl -s ifconfig.me                      # グローバルIPアドレス確認
