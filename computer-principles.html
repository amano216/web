コンピューター原理原則チートシート

プログラミングとコンピューターサイエンスの基本的な原理原則をまとめたリファレンスです。プログラムの基本構造から、データ構造、アルゴリズム、設計原則まで、コンピューターの動作を理解するために必要な概念を網羅しています。

プログラムの基本構造

制御構造の3つの基本パターン

構造 英語名 (English) 説明・用途
順次構造 Sequential Structure 命令を上から順番に実行する最も基本的な構造。
プログラムは記述された順序通りに処理される。
分岐構造 Selection/Conditional Structure 条件によって実行する処理を選択する構造。
if文、switch文、三項演算子などで実現。
反復構造 Iteration/Loop Structure 特定の処理を繰り返し実行する構造。
for文、while文、do-while文などで実現。		 
// 順次構造の例
let x = 10;
let y = 20;
let sum = x + y;
console.log(sum);  // 30

// 分岐構造の例
if (sum > 25) {
    console.log("合計は25より大きい");
} else {
    console.log("合計は25以下");
}

// 反復構造の例
for (let i = 0; i < 5; i++) {
    console.log(i);  // 0, 1, 2, 3, 4
}

データ構造の基本

主要なデータ構造

データ構造 英語名 (English) 特徴・用途
配列 Array 同じ型のデータを連続的に格納。
インデックス（0から始まる数値）でアクセス。
サイズが固定の場合が多い。
リスト List 順序付きのデータコレクション。
動的にサイズ変更可能。
要素の追加・削除が容易。
スタック Stack LIFO（Last In First Out：後入れ先出し）。
最後に追加した要素から取り出す。
関数呼び出しの管理などに使用。
キュー Queue FIFO（First In First Out：先入れ先出し）。
最初に追加した要素から取り出す。
タスク管理やメッセージ処理に使用。
ハッシュテーブル Hash Table キーと値のペアで管理。
高速な検索・挿入・削除が可能。
辞書やマップとも呼ばれる。
ツリー Tree 階層構造を持つデータ構造。
ルートから枝分かれする構造。
ファイルシステムやDOMで使用。
グラフ Graph ノード（頂点）とエッジ（辺）で構成。
ネットワークや関係性の表現に使用。
SNSの友人関係など。		 
// 配列の例
const animals = ["dog", "cat", "sheep"];
console.log(animals[0]);  // "dog"
console.log(animals[2]);  // "sheep"

// スタックの操作例（配列で実装）
const stack = [];
stack.push("A");  // push: 追加
stack.push("B");
stack.push("C");
console.log(stack.pop());  // "C" (LIFO)

// キューの操作例（配列で実装）
const queue = [];
queue.push("First");  // enqueue: 追加
queue.push("Second");
queue.push("Third");
console.log(queue.shift());  // "First" (FIFO)

アルゴリズムの基本

時間計算量（Big O記法）

計算量 名称 説明・例
O(1) 定数時間 データ量に関わらず一定時間。
配列の要素へのアクセス。
O(log n) 対数時間 データ量が倍になっても実行時間はわずかに増加。
二分探索。
O(n) 線形時間 データ量に比例して実行時間が増加。
配列の全要素走査。
O(n log n) 線形対数時間 効率的なソートアルゴリズム。
マージソート、ヒープソート。
O(n²) 二次時間 データ量の二乗に比例。
二重ループ、バブルソート。
O(2ⁿ) 指数時間 データ量が増えると爆発的に増加。
総当たり探索。

基本的なアルゴリズム

アルゴリズム 種類 計算量・特徴
線形探索 探索 O(n) - 先頭から順に探す
二分探索 探索 O(log n) - ソート済み配列で使用
バブルソート ソート O(n²) - 隣接要素を比較交換
クイックソート ソート 平均O(n log n) - 分割統治法
マージソート ソート O(n log n) - 安定ソート

プログラミングパラダイム

主要なプログラミングパラダイム

パラダイム 英語名 特徴・概念
手続き型 Procedural Programming 処理を順番に記述。
関数で処理をまとめる。
C言語、Pascal。
オブジェクト指向 Object-Oriented Programming (OOP) データと処理をオブジェクトにまとめる。
カプセル化、継承、多態性。
Java、C++、Python。
関数型 Functional Programming 関数を組み合わせて処理。
副作用を避ける、不変性。
Haskell、Lisp、JavaScript（一部）。
宣言型 Declarative Programming 「何を」したいかを記述。
「どのように」は記述しない。
SQL、HTML、CSS。

オブジェクト指向の原則（OOP）

原則 英語名 説明
カプセル化 Encapsulation データと処理を1つの単位にまとめ、外部から隠蔽する
継承 Inheritance 既存のクラスから新しいクラスを作成し、機能を受け継ぐ
多態性 Polymorphism 同じインターフェースで異なる実装を提供する
抽象化 Abstraction 複雑な実装の詳細を隠し、必要な部分だけを公開する

設計原則

原則 正式名称 説明
S Single Responsibility Principle
（単一責任の原則）		 クラスは1つの責任だけを持つべき
O Open/Closed Principle
（開放閉鎖の原則）		 拡張に対して開いており、修正に対して閉じている

（リスコフの置換原則）		 派生クラスは基底クラスと置換可能であるべき

（インターフェース分離の原則）		 クライアントは使用しないメソッドに依存すべきではない

（依存性逆転の原則）		 上位モジュールは下位モジュールに依存すべきではない

メモリ管理

メモリ領域の種類

領域 英語名 特徴・用途
スタック Stack ローカル変数、関数の引数を格納。
高速アクセス、自動的に解放。
サイズが限定的。
ヒープ Heap 動的に確保されるメモリ領域。
大きなデータ、オブジェクトを格納。
手動またはGCで解放。
静的領域 Static/Data Segment グローバル変数、静的変数を格納。
プログラム実行中は常に存在。
コード領域 Code/Text Segment 実行可能なプログラムコードを格納。
読み取り専用。

ガベージコレクション（GC）

不要になったメモリを自動的に解放する仕組み。JavaScriptやJava、Pythonなどの言語で採用。プログラマーが明示的にメモリを解放する必要がない。

並行処理と並列処理

並行処理の概念

概念 英語名 説明
プロセス Process 独立したメモリ空間を持つ実行単位。
他のプロセスから隔離されている。
スレッド Thread プロセス内の実行単位。
メモリを共有する軽量な処理単位。
同期処理 Synchronous 処理が順番に実行される。
前の処理が完了するまで待機。
非同期処理 Asynchronous 処理の完了を待たずに次の処理を実行。
コールバック、Promise、async/await。
デッドロック Deadlock 複数のプロセスが互いのリソースを待ち続ける状態。
プログラムが停止する。
競合状態 Race Condition 複数のスレッドが同時に共有データにアクセス。
予期しない結果を生む。		 
// 非同期処理の例（JavaScript）
// Promise
fetch('/api/data')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error(error));

// async/await
async function getData() {
    try {
        const response = await fetch('/api/data');
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error(error);
    }
}

エラー処理

エラーの種類

エラー種類 英語名 説明・例
構文エラー Syntax Error プログラムの文法が間違っている。
括弧の不一致、セミコロンの欠落など。
実行時エラー Runtime Error プログラム実行中に発生。
ゼロ除算、null参照など。
論理エラー Logic Error プログラムは動作するが結果が間違っている。
アルゴリズムの誤りなど。

例外処理

// try-catch-finally の例
try {
    // エラーが発生する可能性のあるコード
    const result = riskyOperation();
    console.log(result);
} catch (error) {
    // エラーが発生した場合の処理
    console.error('エラーが発生しました:', error.message);
} finally {
    // 必ず実行される処理
    cleanup();
}
