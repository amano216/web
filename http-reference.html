HTTPメソッド一覧

HTTP（HyperText Transfer Protocol）で使用される主要なメソッドとステータスコードをまとめました。

HTTPメソッド

主要なHTTPメソッド

メソッド 英語の意味 (Meaning) 日本語の意味・用途
GET Get (取得する) リソースを取得する。サーバーからデータを要求。副作用なし（安全）
POST Post (投稿する) リソースを作成する。データをサーバーに送信。副作用あり
PUT Put (置く) リソース全体を更新・置換する。存在しない場合は作成
PATCH Patch (修正する) リソースの一部を更新する。部分的な変更に使用
DELETE Delete (削除する) リソースを削除する。指定したリソースをサーバーから削除
HEAD Head (頭部) GETと同じだがヘッダーのみ取得。ボディは返さない
OPTIONS Options (選択肢) 対象リソースで利用可能なメソッドを確認。CORSプリフライト
CONNECT Connect (接続する) プロキシサーバーに対してトンネル接続を確立
TRACE Trace (追跡する) リクエストをそのまま返す。デバッグ用（通常は無効）

メソッドの特性

メソッド 安全性
(Safe)		 冪等性
(Idempotent)		 キャッシュ可能
(Cacheable)

POST ✗ いいえ ✗ いいえ △ 条件付き

注：

  • 安全性：サーバーの状態を変更しない
  • 冪等性：同じリクエストを何度実行しても同じ結果になる
  • キャッシュ可能：レスポンスをキャッシュできる

HTTPステータスコード

1xx 情報レスポンス

コード 英語の意味 (Meaning) 日本語の意味・用途
100 Continue 継続。クライアントはリクエストを続行してよい
101 Switching Protocols プロトコル切り替え。WebSocketへの切り替えなど
102 Processing 処理中。長時間かかる処理を実行中
103 Early Hints 早期ヒント。最終レスポンス前のヒント情報

2xx 成功レスポンス

コード 英語の意味 (Meaning) 日本語の意味・用途
200 OK 成功。リクエストが正常に処理された
201 Created 作成完了。新しいリソースが作成された
202 Accepted 受理。リクエストは受理されたが処理は未完了
203 Non-Authoritative Information 信頼できない情報。プロキシが改変した情報
204 No Content 内容なし。成功したがレスポンスボディはない
205 Reset Content 内容リセット。フォームをリセットすべき
206 Partial Content 部分的内容。範囲リクエストの一部を返す

3xx リダイレクション

コード 英語の意味 (Meaning) 日本語の意味・用途
300 Multiple Choices 複数の選択。複数のリソースから選択可能
301 Moved Permanently 恒久的移動。リソースのURLが永続的に変更
302 Found 発見。一時的な移動（元は Moved Temporarily）
303 See Other 他を参照。POSTの結果を別のURLで参照
304 Not Modified 未更新。キャッシュが有効でリソース未変更
307 Temporary Redirect 一時的リダイレクト。メソッドを変更せずリダイレクト
308 Permanent Redirect 恒久的リダイレクト。301と同様だがメソッド変更なし

4xx クライアントエラー

コード 英語の意味 (Meaning) 日本語の意味・用途
400 Bad Request 不正なリクエスト。構文エラーなど
401 Unauthorized 認証が必要。ログインが必要
402 Payment Required 支払いが必要。将来の使用のため予約
403 Forbidden 禁止。アクセス権限がない
404 Not Found 見つからない。リソースが存在しない
405 Method Not Allowed 許可されていないメソッド。HTTPメソッドが無効
406 Not Acceptable 受理できない。要求された形式で提供不可
407 Proxy Authentication Required プロキシ認証が必要
408 Request Timeout リクエストタイムアウト。時間内に完了せず
409 Conflict 競合。リソースの状態と矛盾
410 Gone 消滅。リソースが永久に利用不可
411 Length Required Content-Lengthが必要
413 Payload Too Large ペイロードが大きすぎる
414 URI Too Long URIが長すぎる
415 Unsupported Media Type サポートされていないメディアタイプ
422 Unprocessable Entity 処理できないエンティティ。検証エラー
429 Too Many Requests リクエストが多すぎる。レート制限

5xx サーバーエラー

コード 英語の意味 (Meaning) 日本語の意味・用途
500 Internal Server Error 内部サーバーエラー。サーバー側の一般的エラー
501 Not Implemented 未実装。サーバーが機能を実装していない
502 Bad Gateway 不正なゲートウェイ。上流サーバーから無効な応答
503 Service Unavailable サービス利用不可。一時的な過負荷やメンテナンス
504 Gateway Timeout ゲートウェイタイムアウト。上流サーバーの応答なし
505 HTTP Version Not Supported HTTPバージョン非サポート

HTTPヘッダー

よく使うリクエストヘッダー

ヘッダー 英語の意味 (Meaning) 日本語の意味・用途
Accept Accept 受け入れ可能なメディアタイプ。application/json
Accept-Language Accept Language 優先言語。ja,en;q=0.9
Authorization Authorization 認証情報。Bearer token123
Content-Type Content Type 送信データの形式。application/json
Cookie Cookie クッキー情報。sessionid=abc123
Host Host リクエスト先のホスト名。example.com
Origin Origin リクエスト元のオリジン。CORS用
Referer Referer リクエスト元のURL
User-Agent User Agent クライアントの情報。ブラウザやOS

よく使うレスポンスヘッダー

ヘッダー 英語の意味 (Meaning) 日本語の意味・用途
Content-Length Content Length レスポンスボディのバイト数
Content-Type Content Type レスポンスのメディアタイプ
Set-Cookie Set Cookie クッキーを設定。sessionid=abc123; HttpOnly
Location Location リダイレクト先のURL
Cache-Control Cache Control キャッシュ制御。max-age=3600
Access-Control-Allow-Origin CORS Allow Origin CORS許可オリジン。*または特定のオリジン
ETag Entity Tag リソースのバージョン識別子
Last-Modified Last Modified 最終更新日時
